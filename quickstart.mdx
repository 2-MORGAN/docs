# Déployer en 5 minutes

Ce guide vous accompagne de A à Z pour déployer votre premier service avec CNP.

## Avant de commencer

Vous aurez besoin de :

- Un compte Microsoft `@epita.fr` (ou des identifiants admin fournis par votre encadrant)
- Un dépôt GitLab contenant votre projet (peut être vide, en cours, ou existant)

---

## Étape 1 — Se connecter

Accédez à l'URL CNP fournie par votre encadrant.

<Tabs>
  <Tab title="Connexion Microsoft (recommandé)">
    Cliquez sur **Se connecter avec Microsoft** et authentifiez-vous avec votre compte `@epita.fr`.

    Vous êtes redirigé vers Microsoft, puis automatiquement reconnecté à CNP.
  </Tab>
  <Tab title="Accès admin">
    Saisissez les identifiants fournis par votre encadrant et cliquez sur **Se connecter**.
  </Tab>
</Tabs>

---

## Étape 2 — Connecter votre compte GitLab

La première fois, une bannière jaune s'affiche pour vous demander de configurer votre accès GitLab.

1. Cliquez sur **Configurer le token GitLab**
2. Sur GitLab, allez dans **Profile → Preferences → Access Tokens**
3. Créez un token avec les scopes `api` et `read_user`, et une date d'expiration
4. Copiez le token, collez-le dans CNP avec la date d'expiration
5. Cliquez sur **Enregistrer**

<Tip>
  [Guide détaillé pour créer votre token GitLab →](/guides/pat-setup)
</Tip>

---

## Étape 3 — Déployer un service

1. Cliquez sur **Nouveau service** (bouton en haut à droite du dashboard)
2. Sélectionnez votre dépôt GitLab dans la liste déroulante
3. Le framework est **détecté automatiquement** — vérifiez qu'il est correct, ou choisissez-en un manuellement
4. Cliquez sur **Configurer**

CNP travaille pendant quelques secondes, puis votre service apparaît dans le tableau de bord.

<Warning>
  CNP va ajouter des fichiers dans votre dépôt (`.gitlab-ci.yml`, `Dockerfile`, `k8s/`). Assurez-vous que votre branche par défaut est propre avant de continuer.
</Warning>

---

## Étape 4 — Votre service est en ligne

Votre service apparaît dans le dashboard avec son URL d'accès. Le pipeline GitLab se lance automatiquement.

<Steps>
  <Step title="Pipeline en cours">
    GitLab CI commence à construire votre application. Suivez la progression dans votre projet GitLab → **CI/CD → Pipelines**.
  </Step>
  <Step title="Image Docker publiée">
    Une fois le build terminé, l'image est publiée dans la registry GitLab.
  </Step>
  <Step title="Déploiement Kubernetes">
    Le système de déploiement détecte la nouvelle image et met à jour votre service automatiquement.
  </Step>
  <Step title="Service accessible">
    Accédez à votre service via l'URL affichée dans le dashboard CNP.
  </Step>
</Steps>
