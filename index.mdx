## Qu'est-ce que CNP ?

**CNP (Cloud Native Platform)** est une plateforme développée par l'équipe **2Morgan** pour simplifier le cycle de vie des services cloud-native. En quelques clics, vous connectez un dépôt GitLab existant, choisissez votre framework, et CNP génère automatiquement :

- Un pipeline GitLab CI/CD complet (build → test → Docker)
- Un `Dockerfile` optimisé pour votre stack
- La protection de la branche principale
- Les variables CI nécessaires au registry Docker

## Fonctionnalités principales

<AccordionGroup>
  <Accordion title="Scaffolding automatique">
    CNP pousse directement sur votre dépôt GitLab les fichiers de configuration nécessaires (`.gitlab-ci.yml`, `Dockerfile`, `go.mod`, etc.) générés à partir de templates paramétrés selon votre framework et votre projet.
  </Accordion>

  <Accordion title="Support multi-framework">
    Trois frameworks sont supportés nativement :

    - **Go** — pipeline en 3 stages (build / test / docker), image `golang:alpine`
    - **Next.js** — pipeline en 2 stages (build / docker), cache npm, image `node:alpine`
    - **Spring Boot** — pipeline en 3 stages (test / build / docker), cache Maven, image `maven:3.9-eclipse-temurin-21`
  </Accordion>

  <Accordion title="Intégration GitLab native">
    CNP utilise votre Personal Access Token (PAT) GitLab pour interagir avec vos projets : lecture des dépôts, commit des fichiers de scaffold, protection de branche, injection de variables CI et déclenchement de pipeline.
  </Accordion>

  <Accordion title="Redéploiement en un clic">
    Depuis le dashboard, vous pouvez forcer un nouveau pipeline sur n'importe quel service déjà configuré via le bouton **Redéployer**.
  </Accordion>

  <Accordion title="Infrastructure Kubernetes sur Azure">
    La plateforme elle-même tourne sur un cluster **AKS** (Azure Kubernetes Service) provisionné via Terraform dans la région `switzerlandnorth`.
  </Accordion>
</AccordionGroup>

| Frontend | React 18, TypeScript, Vite, Tailwind CSS |
| Infrastructure | Azure AKS, Terraform |
| CI/CD générée | GitLab CI, Kaniko |
| Registry | GitLab Container Registry |
