## Variables d'environnement

### Obligatoires

Le backend panique au démarrage si ces variables sont manquantes.

| Variable | Description | Exemple |
| --- | --- | --- |
| `PORT` | Port d'écoute du serveur HTTP | `8080` |
| `ADMIN_USERNAME` | Identifiant admin | `admin` |
| `ADMIN_PASSWORD` | Mot de passe admin | `changeme` |
| `JWT_SECRET` | Secret HMAC-SHA256 pour les tokens JWT | `your-256-bit-secret` |
| `FRONTEND_URL` | URL du frontend (pour CORS) | `https://cnp.example.com` |
| `GITLAB_BASE_URL` | URL de base de l'instance GitLab | `https://gitlab.cri.epita.fr` |
| `DATABASE_URL` | URL de connexion PostgreSQL | `postgres://user:pass@host:5432/db` |
| `TOKEN_ENCRYPTION_KEY` | Clé AES pour le chiffrement des tokens (32 octets en hex) | `<64 chars hex>` |
| `MICROSOFT_CLIENT_ID` | App ID Azure pour l'auth SSO | `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` |
| `MICROSOFT_TENANT_ID` | Tenant Azure | `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` |
| `MICROSOFT_ALLOWED_DOMAIN` | Domaine email autorisé | `epita.fr` |

### Optionnelles (avec défauts)

| Variable | Défaut | Description |
| --- | --- | --- |
| `ARGOCD_NAMESPACE` | `argocd` | Namespace ArgoCD dans le cluster |
| `ARGOCD_PROJECT` | `default` | Projet ArgoCD |
| `ARGOCD_DEST_NAMESPACE` | `student-apps` | Namespace cible pour les déploiements |
| `ARGOCD_REPO_SECRET_PREFIX` | `repo-` | Préfixe des secrets de repository ArgoCD |
| `ARGOCD_APP_PREFIX` | `app-` | Préfixe des Applications ArgoCD |
| `KUBECONFIG` | _(défaut kubectl)_ | Chemin vers le kubeconfig |

## Fichier `.env`

```bash
PORT=8080
ADMIN_USERNAME=admin
ADMIN_PASSWORD=supersecret
JWT_SECRET=my-very-long-secret-at-least-32-chars
FRONTEND_URL=http://localhost:5173
GITLAB_BASE_URL=https://gitlab.cri.epita.fr
DATABASE_URL=postgres://cnp:password@localhost:5432/cnpdb
TOKEN_ENCRYPTION_KEY=0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef
MICROSOFT_CLIENT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
MICROSOFT_TENANT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
MICROSOFT_ALLOWED_DOMAIN=epita.fr
```

<Note>
  Les variables déjà définies dans l'environnement ont priorité sur celles du `.env`.
</Note>

## Configuration du frontend

| Variable | Description | Exemple |
| --- | --- | --- |
| `VITE_API_URL` | URL de base du backend CNP | `https://api.cnp.example.com` |

```bash
# cnp-frontend/.env
VITE_API_URL=http://localhost:8080
```

## Générer les clés

```bash
# JWT Secret (32+ chars)
openssl rand -base64 32

# Token Encryption Key (32 bytes = 64 chars hex)
openssl rand -hex 32
```

---

## Base de données

CNP utilise PostgreSQL pour stocker les tokens chiffrés. Le schéma est créé automatiquement au démarrage.

```bash
# Démarrer PostgreSQL avec Docker
docker-compose up -d db

# Ou utiliser le docker-compose complet
docker-compose up -d
```

## Sécurité

<AccordionGroup>
  <Accordion title="JWT Secret">
    Minimum 32 caractères. Les tokens expirent après **24h**.
  </Accordion>

  <Accordion title="Token Encryption Key">
    Clé AES 256 bits (32 octets, encodée en 64 caractères hexadécimaux). Stockez-la dans un secret manager en production.
  </Accordion>

  <Accordion title="Microsoft SSO">
    La validation du token Microsoft utilise les clés JWKS RSA de `login.microsoftonline.com`. Les clés sont mises en cache 1h côté backend.
  </Accordion>

  <Accordion title="CORS">
    Seule l'origine `FRONTEND_URL` est autorisée.
  </Accordion>
</AccordionGroup>
