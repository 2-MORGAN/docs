## Vue d'ensemble

CNP supporte **5 frameworks**. En plus des fichiers CI/CD et Docker, des **manifests Kubernetes communs** (`k8s/`) sont générés pour tous les frameworks.

---

## Manifests Kubernetes (communs)

Générés dans le dossier `k8s/` de votre dépôt pour tous les frameworks :

| Fichier | Description |
| --- | --- |
| `k8s/deployment.yaml` | Déploiement Kubernetes (image, imagePullSecret) |
| `k8s/service.yaml` | Service ClusterIP |
| `k8s/ingress.yaml` | Ingress Traefik avec règles de routage |

Ces fichiers sont surveillés par ArgoCD pour le déploiement automatique.

## Go

**Identifiant** : `go` · **Détection** : `go.mod`, `go.work`

| Fichier | Description |
| --- | --- |
| `.gitlab-ci.yml` | Pipeline 3 stages : build / test / docker |
| `Dockerfile` | Image de production alpine |
| `go.mod` | Module Go avec chemin automatique |
| `.gitignore` | Ignorer les binaires |
| `README.md` | README pré-rempli |

```yaml
stages: [build, test, docker]
# build + test : golang:1.22-alpine
# docker : kaniko → CI_REGISTRY_IMAGE:latest
```

## Next.js

**Identifiant** : `nextjs` · **Détection** : `next.config.*`, `"next"` dans `package.json`

| Fichier | Description |
| --- | --- |
| `.gitlab-ci.yml` | Pipeline 2 stages : build / docker |
| `Dockerfile` | Image multi-stage node:alpine |
| `package.json`, `tsconfig.json`, `next.config.js` | Config projet |
| `app/page.tsx`, `app/layout.tsx` | Pages de base |

```yaml
stages: [build, docker]
# build : node:20-alpine, cache npm, artifacts .next/
# docker : kaniko
```

<Note>
  Utilise `npm ci` sur `main` et les MR, `npm install` sur les autres branches.
</Note>

## Variables CI injectées (tous frameworks)

| Variable | Description |
| --- | --- |
| `REGISTRY_TOKEN` | Token d'accès au registry Docker |
| `REGISTRY_URL` | URL du registry |
| `REGISTRY_USERNAME` | Nom d'utilisateur du registry |

<Warning>
  Les valeurs initiales sont des placeholders. Remplacez-les dans GitLab CI/CD → Variables.
</Warning>

## Build Docker avec Kaniko

Tous les frameworks utilisent [Kaniko](https://github.com/GoogleContainerTools/kaniko). L'image est poussée avec deux tags :

- `CI_REGISTRY_IMAGE:CI_COMMIT_SHORT_SHA` — tag immutable par commit
- `CI_REGISTRY_IMAGE:latest` — tag suivi par ArgoCD
