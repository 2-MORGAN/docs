# Configurer votre accès GitLab

Pour travailler sur vos dépôts, CNP a besoin d'un **Personal Access Token (PAT)** GitLab — un jeton qui lui donne les droits nécessaires pour configurer vos projets en votre nom.

<Info>
  CNP utilise ce token pour lire vos projets, pousser les fichiers de configuration (CI/CD, Docker, Kubernetes), et déclencher les pipelines. Le token est stocké de façon **chiffrée et sécurisée**.
</Info>

---

## Créer le token sur GitLab

<Steps>
  <Step title="Ouvrir les paramètres tokens">
    Connectez-vous à GitLab, puis allez dans votre **profil** (en haut à droite) → **Preferences** → **Access Tokens**.
  </Step>
  <Step title="Créer un nouveau token">
    Cliquez sur **Add new token** et remplissez :

    | Champ | Valeur |
    | --- | --- |
    | Token name | `cnp-platform` (ou le nom de votre choix) |
    | Expiration date | Dans 1 an maximum |
    | Scopes | ✅ `api` et ✅ `read_user` |
  </Step>
  <Step title="Copier le token">
    Cliquez sur **Create personal access token** et **copiez immédiatement** le token — il n'est affiché qu'une seule fois.
  </Step>
</Steps>

---

## Enregistrer le token dans CNP

1. Sur le dashboard CNP, cliquez sur **Configurer le token GitLab** (bannière jaune en haut de page)
2. Collez votre token dans le champ **Token GitLab**
3. Saisissez la **date d'expiration** du token (format `YYYY-MM-DD`)
4. Cliquez sur **Enregistrer**

CNP vérifie immédiatement le token. S'il est valide, votre nom d'utilisateur GitLab s'affiche.

---

## Renouveler un token expiré

Quand votre token expire, la bannière jaune réapparaît. Créez un nouveau PAT sur GitLab et re-enregistrez-le dans CNP — même procédure.

---

## Erreurs courantes

| Message | Cause | Solution |
| --- | --- | --- |
| « Token GitLab invalide » | Token incorrect ou expiré | Créez un nouveau PAT |
| « La date d'expiration est requise » | Champ date vide | Renseignez la date d'expiration |
| « Format de date invalide » | Format incorrect | Utilisez `YYYY-MM-DD` |
| « Impossible de joindre GitLab » | Problème réseau | Vérifiez votre connexion et réessayez |
