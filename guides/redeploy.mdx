# Mettre à jour votre service

## Quand utiliser le redéploiement ?

<Note>
  Pour une mise à jour normale de votre code, un **simple push sur `main` suffit** — le pipeline se déclenche automatiquement. Le bouton **Redéployer** est réservé aux cas particuliers ci-dessous.
</Note>

Utilisez **Redéployer** quand :

- Votre pipeline est bloqué et vous voulez repartir de zéro
- Vous voulez forcer un nouveau déploiement sans modifier votre code
- Les templates CNP ont été mis à jour et vous voulez en bénéficier

---

## Comment redéployer

1. Sur le dashboard CNP, trouvez votre service dans la liste
2. Cliquez sur **Redéployer**

CNP va automatiquement :

1. Re-générer les fichiers de configuration (CI/CD, Docker, Kubernetes)
2. Les pousser sur votre dépôt GitLab
3. Déclencher un nouveau pipeline

---

## Résultat attendu

Le pipeline se relance dans GitLab (visible dans **CI/CD → Pipelines**). Une fois terminé, votre service est mis à jour avec la dernière version de votre code.

---

## Problèmes courants

| Problème | Solution |
| --- | --- |
| Pipeline échoue après redéploiement | Consultez les logs GitLab pour l'erreur précise |
| Service n'apparaît plus | Attendez quelques minutes — le déploiement peut prendre du temps |
| Erreur « framework non reconnu » | Le tag de framework a été retiré de GitLab — reconfigurez le service |

<Tip>
  Si le pipeline échoue, consultez l'onglet [Aide et dépannage](/guides/aide) pour les erreurs courantes.
</Tip>
