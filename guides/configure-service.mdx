## Prérequis

- Être connecté à CNP
- Avoir [configuré votre PAT GitLab](/guides/pat-setup)
- Avoir un dépôt GitLab existant (peut être vide ou contenir du code)

---

## Depuis le dashboard

1. Cliquez sur **Nouveau service** (en haut à droite du dashboard)
2. La modale **Configurer un service** s'ouvre

### Sélectionner le dépôt

Le champ affiche tous vos projets GitLab dont vous êtes membre (jusqu'à 100 par page).

### Choisir le framework

Le framework peut être :

**Détection automatique** (recommandé) — laissez le champ vide. CNP analyse les fichiers du dépôt :

| Fichier | Framework détecté |
| --- | --- |
| `go.mod` / `go.work` | Go |
| `next.config.*` | Next.js |
| `nest-cli.json` | NestJS |
| `manage.py` | Django |
| `pom.xml` → spring-boot | Spring Boot |

**Sélection manuelle** — choisissez parmi :

| Framework | Stack |
| --- | --- |
| `go` | Service backend Go |
| `nextjs` | Application Next.js TypeScript |
| `nestjs` | API NestJS TypeScript |
| `springboot` | Service Java Spring Boot |
| `django` | Application Python Django |

### Valider

Cliquez sur **Configurer**. CNP exécute séquentiellement :

1. Récupère les infos du projet GitLab
2. Détecte le framework (si non fourni)
3. Génère les fichiers via les templates (CI/CD \+ Docker \+ K8s manifests)
4. Pousse les fichiers sur la branche par défaut
5. Configure la protection de branche
6. Ajoute les topics GitLab
7. Injecte les variables CI
8. Crée les deploy tokens (repo \+ registry)
9. Synchronise ArgoCD (secret repo, secret registry, Application)

## Gestion des erreurs

| Code | Message | Action |
| --- | --- | --- |
| `409 Conflict` | Ce repository est déjà configuré | Vérifiez l'état du dépôt |
| `403 Forbidden` | Token GitLab non configuré | [Configurez votre PAT](/guides/pat-setup) |
| `400` | framework non reconnu, selection manuelle requise | Spécifiez le framework manuellement |
| `500` | repository configuré mais enregistrement de déploiement échoué | ArgoCD indisponible — réessayez avec redeploy |

## Après la configuration

<Checklist>
  * [ ] Le service apparaît dans le dashboard avec son URL externe
  * [ ] Le dépôt GitLab contient `.gitlab-ci.yml`, `Dockerfile` et `k8s/`
  * [ ] Les topics `cnp-2morgan` et le framework sont visibles sur le projet
  * [ ] Un premier pipeline CI est en cours
  * [ ] L'application ArgoCD est visible dans le dashboard ArgoCD
</Checklist>

## Remplacer les placeholders de registry

Dans GitLab **Settings → CI/CD → Variables** :

```text
REGISTRY_URL      → registry.cri.epita.fr  (ou votre registry)
REGISTRY_USERNAME → votre nom d'utilisateur GitLab
REGISTRY_TOKEN    → votre PAT avec scope read_registry / write_registry
```
