# Déployer un service

## Prérequis

- Être connecté à CNP ([guide connexion →](/guides/connexion))
- Avoir configuré votre accès GitLab ([guide token →](/guides/pat-setup))
- Avoir un dépôt GitLab avec votre code (vide ou existant)

---

## Lancer le déploiement

1. Sur le dashboard, cliquez sur **Nouveau service**
2. **Sélectionnez votre dépôt GitLab** dans la liste déroulante
3. Le **framework est détecté automatiquement** — vérifiez qu'il est correct

Si la détection est incorrecte ou indisponible, sélectionnez manuellement :

| Framework | Choisir si votre projet utilise... |
| --- | --- |
| `go` | Go |
| `nextjs` | Next.js / React |
| `nestjs` | NestJS (Node.js) |
| `springboot` | Java / Spring Boot |
| `django` | Python / Django |

4. Cliquez sur **Configurer**

---

## Ce qui se passe

CNP configure votre service automatiquement :

<Steps>
  <Step title="Analyse du projet">
    CNP récupère les informations de votre dépôt et détecte le framework si non spécifié.
  </Step>
  <Step title="Génération des fichiers">
    Les fichiers sont générés et poussés sur votre branche par défaut :

    - `.gitlab-ci.yml` — pipeline CI/CD
    - `Dockerfile` — image de votre application
    - `k8s/` — fichiers de déploiement Kubernetes
  </Step>
  <Step title="Configuration GitLab">
    CNP protège votre branche principale et configure les accès nécessaires au déploiement.
  </Step>
  <Step title="Déploiement sur Kubernetes">
    Votre service est enregistré et le pipeline CI/CD démarre automatiquement.
  </Step>
</Steps>

<Warning>
  CNP pousse des fichiers directement sur votre branche par défaut. Si vous avez des modifications non commitées, committez-les d'abord pour éviter les conflits.
</Warning>

---

## Résultat

Votre service apparaît dans le dashboard avec :

- ✅ Son **URL d'accès** (disponible dès que le premier pipeline a terminé)
- ✅ Son **framework** détecté
- ✅ Un lien vers votre **dépôt GitLab**

<Tip>
  La première URL peut prendre quelques minutes, le temps que le pipeline CI/CD termine son premier build.
</Tip>

---

## Erreurs possibles

| Message | Cause | Solution |
| --- | --- | --- |
| « Ce repository est déjà configuré » | Service déjà existant | Utilisez [Mettre à jour](/guides/redeploy) si vous voulez le redéployer |
| « Token GitLab non configuré » | PAT absent | [Configurez votre accès GitLab](/guides/pat-setup) |
| « Framework non reconnu » | Détection impossible | Sélectionnez le framework manuellement |
